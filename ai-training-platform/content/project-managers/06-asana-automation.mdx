## Technique 1: Google Chat to Asana (The Smart Way)

Stop copy-pasting! Automate it!

### The Old Way (Why It's Painful)

**Traditional workflow:**
1. Someone sends a message in Google Chat: "We need to fix the login bug"
2. You copy the message
3. Open Asana
4. Create a new task
5. Paste the message
6. Add details manually
7. Assign it
8. Set due date
9. Repeat for every message

**Time:** 5 minutes per task Ã— 20 tasks = 100 minutes wasted!

### The New Way (Automated)

**Automated workflow:**
1. Someone sends a message in Google Chat
2. You star the message (or it auto-triggers)
3. AI parses the message automatically
4. Asana task is created automatically
5. Done!

**Time:** 30 seconds per task Ã— 20 tasks = 10 minutes total!

### How to Set It Up

**Tools needed:**
- Zapier or Make (automation platforms)
- Google Chat
- Asana
- AI (ChatGPT API or similar)

**The workflow:**
1. **Trigger:** When you star a message in Google Chat
2. **AI Step:** Parse the message:
   > "Read this chat message. Identify the core task. If it's a bug, format the title as '[BUG] - {Issue}'. If it's a feature, format as '[FEAT] - {Feature}'. Extract the due date if mentioned (default to next Friday if not). Extract the assignee if mentioned."
3. **Action:** Create Asana task with parsed information

**What this saves:** Hours of manual data entry!

## Technique 2: The "Zombie Task" Hunter (Cleaning Up Boards)

Projects get bloated with tasks nobody intends to do. AI can find them!

### What Are Zombie Tasks?

**Zombie tasks** = Tasks that:
- Haven't been updated in 30+ days
- Aren't marked complete
- Nobody is working on
- Clog up your board
- Make it hard to see what's actually important

### How to Find Them

**Step 1:** Export your Asana project as CSV

**Step 2:** Upload to ChatGPT or Claude

**Step 3:** Ask AI:
> "Analyse this list of 500 tasks. Identify tasks that haven't been updated in 30 days but are not marked complete. Suggest which ones should be archived based on their low priority or lack of activity. Group them by: Definitely archive, Maybe archive, Keep active."

**Step 4:** Review AI's suggestions

**Step 5:** Archive the zombie tasks

**What this saves:** Hours of manually reviewing hundreds of tasks!

![Zombie Tasks](https://media.giphy.com/media/l0HlNQ03J5JxX6lva/giphy.gif)
*'Your Board: 500 tasks, most are zombies. AI: Let me find the dead ones. Result: 200 tasks to archive. You: Finally, a clean board!'*

### Example: Zombie Task Analysis

**AI analysis:**
> "Zombie Tasks Found:
> 
> **Definitely Archive (150 tasks):**
> - Tasks from 6+ months ago with no updates
> - Low priority tasks with no assignee
> - Tasks marked 'On Hold' for 90+ days
> 
> **Maybe Archive (50 tasks):**
> - Tasks with recent comments but no progress
> - Tasks assigned to people who left the company
> 
> **Keep Active (300 tasks):**
> - Tasks updated in last 30 days
> - High priority tasks
> - Tasks with active assignees"

**See?** AI organised your board for you!

## Technique 3: Brief to Subtasks (Breaking Down Projects)

Paste a client email or PDF brief. Get subtasks automatically!

### The Problem

**Common scenario:**
- Client sends a brief: "We need a new website with 5 pages, contact form, and blog"
- You have to break it down manually
- Create 20+ subtasks
- Organise them by phase
- Estimate hours
- Takes forever

### The Solution

**Step 1:** Paste the client brief (email or PDF)

**Step 2:** Ask AI:
> "Break this project down into Asana subtasks. Group them by phase (Design, Development, QA). Estimate hours for each. Include dependencies. Output as a CSV format I can import directly into Asana."

**Step 3:** Import the CSV into Asana

**Step 4:** Review and adjust

**What this saves:** Hours of manual task creation!

### Example: Brief to Subtasks

**Client brief:**
> "We need a new website with homepage, about page, services page, contact form, and blog."

**AI-generated subtasks:**
```csv
Task,Phase,Estimated Hours,Dependencies
Design homepage mockups,Design,8,None
Design about page,Design,6,Homepage approved
Design services page,Design,6,Homepage approved
Develop homepage,Development,16,Homepage designs approved
Develop about page,Development,12,About designs approved
Develop services page,Development,12,Services designs approved
Build contact form,Development,8,None
Set up blog,Development,10,None
QA all pages,QA,8,All development complete
```

**See?** Complete task breakdown in seconds!

## Your Practice Exercise: The Board Cleanup

Ready to clean up a messy board? Let's do it!

### Step 1: Export Your Board (5 minutes)

**What to do:**
1. Go to your messiest Asana project
2. Export as CSV
3. Download the file

**Don't have a messy board?** That's okay! Create a sample one with 20-30 tasks for practice.

### Step 2: Analyse with AI (10 minutes)

**What to do:**
1. Upload the CSV to ChatGPT or Claude
2. Ask AI:
   > "Analyse this Asana project. Find the 'bottleneck' (the person with the most overdue tasks). Identify zombie tasks (not updated in 30+ days). Suggest which tasks should be archived."
3. Review the analysis

### Step 3: Clean Up (15 minutes)

**What to do:**
1. Archive zombie tasks AI identified
2. Reassign overdue tasks if needed
3. Update task statuses
4. Organise remaining tasks

### Step 4: Automate (Optional, 15 minutes)

**What to do:**
1. Set up a rule (or draft the logic):
   - Auto-move tasks to "Done" when a checkbox is ticked
   - Auto-assign tasks based on keywords
   - Auto-set due dates based on task type
2. Test the automation
3. Refine if needed

### Questions to Reflect On

- Did AI correctly identify the bottleneck?
- How much time would it take to manually clean that board?
- Are your task titles clear enough for AI to understand?
- What other automations could help?

## Common Mistakes (And How to Avoid Them)

### Mistake #1: Over-Automating

**The problem:** Automating everything, even things that need human judgement

**The solution:** Automate repetitive tasks. Keep human judgement for important decisions.

### Mistake #2: Not Testing Automations

**The problem:** Setting up automation and assuming it works

**The solution:** Always test automations. Start small, then expand.

### Mistake #3: Vague Task Titles

**The problem:** "Fix bug" is too vague for AI to understand

**The solution:** Use clear, descriptive task titles. AI works better with specific information.

### Mistake #4: Not Cleaning Boards Regularly

**The problem:** Boards get messy, then it's overwhelming to clean

**The solution:** Clean boards monthly. Use AI to make it fast!
